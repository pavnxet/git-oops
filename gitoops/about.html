About Git Oops

Git Oops is a free, open-source tool designed to help developers recover from common Git mistakes without losing their minds—or their code.

Git's official documentation is great for learning Git concepts, but terrible for recovery. When something goes wrong, developers search Stack Overflow, get conflicting answers from 2012, and often pick the wrong one. Git Oops provides a single, trusted, visual reference that gives you the exact command for your situation, complete with warnings for destructive operations.

Philosophy

Keep it dead simple. The site uses pure HTML, CSS, and Vanilla JS. There's no build step, no npm dependencies, and no framework. It works offline, loads instantly, and is trivially easy to contribute to.

How to Contribute

The "database" of Git Oops is just a single JSON file. If you know a common Git mistake that isn't listed, or if you spot an error, we'd love your help!

  1. Fork the repository on GitHub.
  2. Edit data/scenarios.json to add or update a scenario.
  3. Submit a Pull Request.

Scenario JSON Schema

When adding a new scenario, follow this format:

{
  "id": "unique-slug-name",
  "title": "Short descriptive title",
  "category": "commits|branches|remote|history|stash|merge|rebase",
  "tags": ["keywords", "for", "search"],
  "description": "Plain English description of the problem.",
  "danger": "safe|caution|destructive",
  "steps": [
    {
      "condition": "Optional condition if there are multiple ways (e.g., 'If pushed')",
      "commands": ["git command 1", "git command 2"],
      "explanation": "What this does.",
      "warning": "Optional warning text for dangerous actions."
    }
  ],
  "related": ["id-of-related-scenario"]
}
